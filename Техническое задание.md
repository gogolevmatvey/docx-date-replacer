# Техническое задание: Замена даты в документах .docx

## 1. Общие сведения

**Наименование проекта:** Docx Date Replacer  
**Назначение:** Автоматическая замена даты в блоке «УТВЕРЖДАЮ» в документах формата .docx  
**Версия:** 2.0

## 2. Область применения

Проект предназначен для пакетной обработки документов формата .docx, содержащих блок утверждения директора института с датой.

### 2.1. Целевые документы

Документы содержат блок вида:
```
УТВЕРЖДАЮ:
Директор института
___________Паньшин И.А.
«29» января 2026 г.
```

### 2.2. Структура входных данных

- Корневая папка: `090303/`
- Подпапки по дисциплинам (например, `Б1.В.01 Основы военной подготовки/`)
- Файлы .docx различных типов: ФОМ, Ann, RP и др.

## 3. Требования к функциональности

### 3.1. Основные функции

1. **Поиск документов**
   - Рекурсивный обход папки `090303/`
   - Фильтрация файлов по расширению `.docx`
   - Исключение временных файлов (начинающихся с `~$`)

2. **Распознавание даты**
   - Поиск **только** даты «29» января 2026 г.
   - Другие даты не обрабатываются (например, «27» февраля 2026 г.)
   - Обработка различных форматов написания:
     - `«29» января 2026 г.` (стандартный)
     - `«29» января 2026г.` (без пробела перед «г.»)
     - `«29»  января  2026 г.` (с множественными пробелами)
     - `« 29 » января 2026 г.` (с пробелами внутри кавычек)

3. **Замена даты**
   - Заменяемая дата: **«29» января 2026 г.** → **«26» февраля 2026 г.**
   - Обработка **только на первой странице** документа
   - Сохранение исходного форматирования:
     - Шрифт Times New Roman
     - Размер шрифта 12pt
     - Все стили и форматирование

4. **Сохранение результатов**
   - Оригинальные файлы не модифицируются
   - Все файлы копируются в папку `090303_new/`
   - Файлы с датой «29» января 2026 г. — дата заменяется
   - Файлы без даты или с другой датой — копируются без изменений
   - Сохранение оригинальной структуры подпапок

### 3.2. Дополнительные функции

- Вывод статистики обработки
- Логирование процесса обработки
- Обработка ошибок (повреждённые файлы, отсутствие доступа)
- Покрытие тестами (34 юнит- и интеграционных теста)

### 5.2. Компоненты системы

#### 5.2.1. Модуль `config.py`

**Конфигурационные параметры:**
```python
SOURCE_FOLDER = "090303"
OUTPUT_FOLDER = "090303_new"
NEW_DATE = "«26» февраля 2026 г."
DATE_PATTERN = r'«29»\s+(января)\s+(2026)\s*г\.?'
FIRST_PAGE_PARAGRAPHS = 50
```

#### 5.2.2. Модуль `date_replacer.py`

**Назначение:** Поиск и замена даты «29» января 2026 г.

**Основные методы:**
- `find_date(text)` — поиск даты в тексте
- `find_date_details(text)` — получение деталей (день, месяц, год)
- `replace_date(text)` — замена даты
- `validate_date_format(date_string)` — валидация формата
- `has_approval_block(text)` — проверка блока «УТВЕРЖДАЮ»
- `find_date_in_first_paragraphs(doc)` — поиск на первой странице

#### 5.2.3. Модуль `docx_processor.py`

**Назначение:** Обработка документов .docx

**Основные методы:**
- `load_document(filepath)` — загрузка документа
- `save_document(doc, filepath)` — сохранение документа
- `process_paragraphs(doc)` — обработка параграфов (первая страница)
- `process_tables(doc)` — обработка таблиц (первая страница)
- `process_document(input_path, output_path)` — полная обработка
- `find_docx_files(root_folder)` — поиск файлов .docx
- `copy_folder_structure(src_root, dst_root)` — копирование структуры папок
- `get_full_text(doc)` — получение полного текста (включая таблицы)

### 5.3. Алгоритм обработки

```
1. Инициализация
   └─> Загрузка конфигурации из config.py

2. Поиск файлов
   └─> Рекурсивный обход папки 090303/
   └─> Фильтрация .docx файлов
   └─> Исключение временных файлов (~$*)

3. Обработка каждого файла
   ├─> Загрузка документа
   ├─> Поиск даты «29» января 2026 г. на первой странице
   ├─> Если дата найдена:
   │   ├─> Замена даты в параграфах (с сохранением шрифта 12pt)
   │   ├─> Замена даты в таблицах (с сохранением шрифта 12pt)
   │   └─> Сохранение в 090303_new/ с новой датой
   └─> Если дата не найдена:
       └─> Копирование в 090303_new/ без изменений

4. Завершение
   └─> Вывод статистики
   └─> Логирование результатов
```

## 6. Форматы данных

### 6.1. Входные данные

- Папка с документами: `090303/`
- Формат файлов: `.docx` (Office Open XML)
- Ожидаемая дата: `«29» января 2026 г.` (различные форматы написания)

### 6.2. Выходные данные

- Папка вывода: `090303_new/` с сохранением структуры подпапок
- Файлы с заменённой датой: `«26» февраля 2026 г.`
- Файлы без изменений: оригинальное содержимое

### 6.3. Поддерживаемые форматы даты

**Входные (найдены и заменены):**
- `«29» января 2026 г.`
- `«29» января 2026г.`
- `«29»  января  2026 г.`
- `« 29 » января 2026 г.`

**Не обрабатываются:**
- `«27» февраля 2026 г.` (другая дата)
- `«29» января 2025 г.` (другой год)
- `«28» января 2026 г.` (другой день)

## 7. Обработка ошибок

| Ошибка | Действие |
|--------|----------|
| Файл не найден | Пропуск, запись в лог |
| Файл повреждён | Пропуск, запись в лог |
| Нет прав доступа | Пропуск, запись в лог |
| Дата «29» января 2026 г. не найдена | Копирование без изменений |
| Ошибка записи | Пропуск файла, запись в лог |
